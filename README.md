NSL-KDD: Mathematically Optimized Dataset for AIThis repository provides a model-ready version of the NSL-KDD dataset. I have transformed the raw network traffic data into a purely numerical format so that any Machine Learning model can process it immediately without further cleaning.🛠️ The 4 Core Processing Steps1. Data Ingestion & CleaningSchema Assignment: Raw NSL-KDD files do not have headers. I manually assigned 43 column names to identify features like duration, protocol_type, and src_bytes.Metadata Removal: I dropped the difficulty_level column.Why? This column is a human-assigned score. Including it would allow the AI to "cheat" by looking at the difficulty instead of learning the actual network patterns.2. Attack Family Mapping (Simplification)The Change: I took over 30 specific sub-attacks (like neptune, satan, or smurf) and mapped them into 5 major categories: DoS, Probe, R2L, U2R, and Normal.The "0 to 4" Logic: To make these usable for math, I converted these families into integers:0: DoS1: Probe2: R2L3: U2R4: Normal3. Categorical Encoding (Words to Numbers)The Change: Features like protocol_type (TCP, UDP, ICMP) and service (HTTP, FTP) were originally text.The Mathematical Fix: I used LabelEncoder to turn these columns (protocol_type, service, flag, label) into unique integers.Why? Machine Learning models are essentially large calculators. They cannot multiply "TCP" by a weight, but they can multiply by "1".4. Min-Max Scaling (The "0 to 1" Transformation)The Change: I applied a MinMaxScaler to every numerical feature.The Math: This squashes every value into a range between 0 and 1 using the formula:$$x_{scaled} = \frac{x - x_{min}}{x_{max} - x_{min}}$$Result: Preprocessing is complete with no more words in the mathematical features.📂 Dataset StatisticsFeatures: 41 numerical columns (post-cleaning).Training Set: 100,778 rows (80% of data).Testing Set: 25,195 rows (20% of data).Reproducibility: I used random_state=42 to ensure the mathematical split remains the same every time it is run.
